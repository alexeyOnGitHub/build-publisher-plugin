Select this box to add test result summaries to the outgoing messages. Note that only "published" test results will be found by the plugin.
  • Native Maven build jobs automatically publish test results, no additional configuration necessary.
  • Freestyle jobs require additional configuration: need to add a Post-Build step to your Build Job(s) called "Publish JUnit test results report" (provided by JUnit Plugin) with something like "test-reports/*.xml" in "Test report XMLs" field (depending on where your custom job saves results).

Sample JSon element that will be added to the message: 

  "test_results": {
        "total_count": 13,
        "skip_count": 0,
        "fail_count": 3,
        "failed_tests": [
            "mychecks.MyServerTest--image--test_health_with_server_status_url",
            "mychecks.MyServerTest--www4--test_health_with_server_status_url"
        ]
    }
